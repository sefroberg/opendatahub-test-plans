# TC-E2E-001: Full tenant lifecycle — provision, use API key, delete

**Objective**: Verify the complete lifecycle of a tenant:
provisioning, API key creation and validation, and full
cleanup on deletion.

**Preconditions**:

- OpenShift 4.19+ cluster with RHOAI 3.4+, MaaS operator, and Kuadrant deployed
- Cluster admin credentials available
- At least one MaaSModelRef CR registered in the cluster

**Test Steps**:

1. Create Tenant CR `tenant-e2e` as cluster admin
2. Verify namespace `tenant-e2e` is created with `maas.opendatahub.io/tenant=tenant-e2e` label
3. Create MaaSSubscription and MaaSAuthPolicy in namespace `tenant-e2e`
4. Verify Kuadrant AuthPolicy is generated on the shared Gateway
5. Create an API key via `POST /v1/api-keys` authenticated as tenant-e2e
6. Validate the API key via `POST /internal/v1/api-keys/validate` — verify `tenant=tenant-e2e` in response
7. Delete Tenant CR `tenant-e2e`
8. Verify namespace, MaaSSubscription, MaaSAuthPolicy, and Kuadrant AuthPolicy are all deleted
9. Verify the API key from step 5 no longer validates (returns 401)

**Expected Results**:

- Steps 1–4: Namespace and policies provisioned successfully
- Step 6: `ValidationResult` contains `"tenant": "tenant-e2e"`
- Steps 8–9: All resources deleted, API key invalidated
