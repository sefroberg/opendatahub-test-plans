# TC-E2E-002: Multi-tenant isolation — two tenants cannot cross-access each other's resources

**Objective**: Verify end-to-end isolation between two
simultaneously active tenants across API keys,
subscriptions, and webhook enforcement.

**Preconditions**:

- OpenShift 4.19+ cluster with RHOAI 3.4+, MaaS operator, Kuadrant, and admission webhook deployed
- Cluster admin credentials available

**Test Steps**:

1. Provision `tenant-alpha` and `tenant-beta` with separate MaaSSubscriptions and MaaSAuthPolicies
2. Create an API key for each tenant
3. Validate tenant-alpha's key — verify `tenant=tenant-alpha` in `ValidationResult`
4. Validate tenant-beta's key — verify `tenant=tenant-beta` in `ValidationResult`
5. Attempt to use tenant-alpha's key to access tenant-beta's endpoint — verify `401`
6. Attempt to use tenant-beta's key to access tenant-alpha's endpoint — verify `401`
7. Call `GET /v1/api-keys` as tenant-alpha — verify no tenant-beta keys in response
8. Attempt to create a MaaSSubscription in an unlabeled namespace — verify webhook rejects it

**Expected Results**:

- Steps 3–4: Each `ValidationResult` contains the correct `tenant` value
- Steps 5–6: Cross-tenant requests return `401 Unauthorized`
- Step 7: Only tenant-alpha's keys are returned
- Step 8: Webhook returns a rejection error with a clear message
