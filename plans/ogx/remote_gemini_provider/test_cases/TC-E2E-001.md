# TC-E2E-001: Deploy OGX with Gemini and run chat completion

**Objective**: Validate the complete user journey from deploying
an OGX distribution with `remote::gemini` provider to
successfully running a Gemini chat completion.

**Preconditions**:

- OpenShift cluster with RHOAI 3.5 EA2+ deployed
- OGX K8s Operator installed
- Valid Gemini API key available

**Test Steps**:

1. Create a Kubernetes Secret with the Gemini API key
2. Create a `LlamaStackDistribution` CR referencing the secret
   to inject `GEMINI_API_KEY`
3. Wait for the OGX pod to reach Running state
4. Send GET `/v1/providers` and verify `remote::gemini` is
   listed
5. Verify the distribution's `config.yaml` uses
   `provider_type: remote::gemini` without `network.headers`
6. Send POST `/v1/chat/completions` with a Gemini model,
   `temperature: 0.7`, and `stream: false`
7. Verify the response is a valid chat completion
8. Send the same request with `stream: true`
9. Verify streaming SSE chunks are delivered and terminate
   with `[DONE]`

**Expected Results**:

- OGX pod starts successfully with `GEMINI_API_KEY` injected
- `/v1/providers` returns `remote::gemini` as an active provider
- Non-streaming chat completion returns HTTP 200 with valid
  `choices[0].message.content`
- Streaming chat completion returns chunked SSE data ending
  with `[DONE]`
- No `network.headers` workaround is required in the config
