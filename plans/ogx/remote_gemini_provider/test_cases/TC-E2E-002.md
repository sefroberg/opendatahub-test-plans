# TC-E2E-002: Full tool calling workflow with Gemini

**Objective**: Validate the complete tool calling workflow through
the `remote::gemini` provider, including tool definition,
invocation, and result handling.

**Preconditions**:

- OGX distribution deployed with `remote::gemini` provider active
- Valid `GEMINI_API_KEY` configured

**Test Steps**:

1. Send POST `/v1/chat/completions` with a tool definition
   (`get_weather` function) and a user message that triggers
   tool invocation ("What is the weather in Paris?")
2. Parse the response to extract `tool_calls` from the
   assistant message
3. Construct a follow-up request with the tool result in the
   messages array including the `tool_call_id` from step 2
   (role: tool, tool_call_id: `<id>`, content: weather data)
4. Send the follow-up request to `/v1/chat/completions`
5. Verify the model incorporates the tool result into its
   final response

**Expected Results**:

- First request returns `tool_calls` with
  `function.name: get_weather` and valid `arguments`
- `finish_reason` is `tool_calls` on the first response
- Second request (with tool result) returns HTTP 200 with
  the model's final answer incorporating the weather data
- The complete tool calling round-trip succeeds without errors

**Test Data**:

```json
{
  "model": "<GEMINI-MODEL-ID>",
  "messages": [
    {"role": "user", "content": "What is the weather in Paris?"}
  ],
  "tools": [
    {
      "type": "function",
      "function": {
        "name": "get_weather",
        "description": "Get the current weather",
        "parameters": {
          "type": "object",
          "properties": {
            "location": {"type": "string"}
          },
          "required": ["location"]
        }
      }
    }
  ]
}
```
