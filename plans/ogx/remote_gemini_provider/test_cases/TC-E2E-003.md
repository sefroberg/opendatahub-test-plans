# TC-E2E-003: Embedding generation end-to-end with Gemini

**Objective**: Validate the complete embedding generation workflow
through the `remote::gemini` provider, including proper handling
of missing usage statistics.

**Preconditions**:

- OGX distribution deployed with `remote::gemini` provider active
- Valid `GEMINI_API_KEY` configured

**Test Steps**:

1. Verify `remote::gemini` is active via GET `/v1/providers`
2. Send a POST to the embeddings endpoint with a short text
   input
3. Verify the response contains valid embedding vectors
4. Send a POST with a longer multi-sentence text input
5. Verify the response contains valid embedding vectors
6. Check whether the response includes `usage` statistics;
   verify the request succeeds regardless

**Expected Results**:

- `/v1/providers` confirms `remote::gemini` is active
- Both embedding requests return HTTP 200
- Response `data` array contains entries with `embedding`
  arrays of floating-point numbers
- No `AttributeError` or HTTP 500 errors occur even if
  Gemini omits `usage` in the response
- Embedding vectors have consistent dimensions across
  requests
